3. What to Build — Detailed Breakdown

3.1 Project Setup

  • Initialize Next.js project with TypeScript: npx create-next-app@latest stockwiz-web --typescript --tailwind --app
  • Install shadcn/ui: npx shadcn-ui@latest init
  • Install dependencies: npm install firebase @tanstack/react-query axios
  • Configure Firebase Auth (Google provider — same Firebase project as Android app)
  • Set up environment variables for API base URL

3.2 API Integration Layer

  • Create an API client (axios wrapper) pointing to the backend
  • TypeScript interfaces matching Kotlin data classes:
    • ScanResultItem — ticker, price, rsi, beta, csps, diagonals, verticals, longLeaps, levels, signals, etc.
    • CspResult — strike, premium, delta, roc, bt, expiry
    • DiagonalResult — longLeg, shortLeg, netDebt, yieldRatio, bt, expiry
    • VerticalResult — strikes, netDebit, bt, expiry
    • LongLeapsResult — expiry, strike, premium, leverage, intrinsicBuffer, bt
    • StockLevels — atr, support, resistance, swingHigh60d, swingLow60d, high52w, stopLoss, target, riskReward, riskNote
    • ActivePosition, ClosedPosition, TradeEntry, BacktestRequest/Response
    • HealthResponse — status, capitalHealth, performance, activePositions, closedPositions
  • Auth interceptor to attach Bearer token to all requests
  • Same endpoints: /scan, /backtest, /portfolio/*, /health

3.3 Scan Page (Home /)

  • Watchlist editor (stored in localStorage or user account)
  • Strategy dropdown: All, CSPs, Diagonals, Verticals, Long LEAPS
  • Tuner dialog: target delta, min ROC
  • Manual ticker search bar
  • Results cards with:
    • Ticker + Price header
    • RSI / Beta / IV chips (color-coded by value range)
    • Recommendation badge (BUY/SELL/HOLD) + overall assessment badge
    • Stock summary text
    • Expandable "More details": SMA200, Off 52W High, bullish/bearish signals, Key Levels card (deduplicated)
    • Strategy results: CSP, Diagonal, Vertical, LEAPS rows with BT badge + "Add to Portfolio" button
  • Batch scanning with progress indicator (batches of 5 tickers)
  • Sort by strategy metric (ROC, yield, etc.)

3.4 Portfolio Page (/portfolio)

  • Active positions table (sortable columns)
  • Closed positions table
  • Capital health + performance metrics cards
  • "Add Manual Position" dialog with fields: ticker, strategy, strike, expiry, premium, contracts
  • Edit position dialog
  • Close position dialog (exit price, exit date)
  • Delete position with confirmation
  • Auto-add from scan results
  • Local storage cache + backend sync (same merge logic as Android)

3.5 AI Guru Page (/ai-guru)

  • Ticker input
  • Strategy selector: CSP, Sell Call, Vertical, Diagonal, Long LEAPS
  • Dynamic input fields per strategy:
    • CSP / Sell Call / Long LEAPS: Strike, Expiry, Premium
    • Vertical: Strike Buy, Strike Sell, Expiry, Premium
    • Diagonal: Strike Buy, Strike Sell, Expiry Buy, Expiry Sell, Premium
  • Backtest results display: win rate, avg return, risk profile

3.6 Alerts Page (/alerts)

  • Notification history list (stored per user in backend or localStorage)
  • Notification cards with timestamp, ticker, strategy, details
  • Mark as read / clear all

3.7 Authentication

  • Google Sign-In button using Firebase Auth JS SDK
  • Protected routes (redirect to /login if not authenticated)
  • Token passed to API calls via axios interceptor
  • Same Firebase project = same user accounts as Android app
  • User profile display (name, email, avatar from Google)

3.8 Responsive Design

  • Mobile-first layouts with Tailwind breakpoints (sm:, md:, lg:)
  • Bottom navigation on mobile, sidebar on desktop
  • Touch-friendly buttons and cards
  • Tables → card lists on mobile
  • PWA (Progressive Web App) for "app-like" feel on mobile home screen

10. Deployment Steps

  1. Create a new GitHub repo: StockWizAI-Web
  2. Push the Next.js project to it
  3. Connect to Vercel → auto-deploys on every push
  4. Set environment variables in Vercel dashboard:
    • NEXT_PUBLIC_API_URL=https://financestreamai-backend.onrender.com/api/v1
    • NEXT_PUBLIC_FIREBASE_API_KEY=...
    • NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
    • NEXT_PUBLIC_FIREBASE_PROJECT_ID=...
  5. Add CORS to backend for the Vercel domain
  6. (Optional) Custom domain: stockwiz.ai or app.stockwiz.ai
