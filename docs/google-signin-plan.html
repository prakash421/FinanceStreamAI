StockWiz AI — Google Sign-In Implementation Plan

Date: April 18, 2026
Goal: Add Google Sign-In using Firebase Auth to support per-user data storage, enabling future features like Plaid brokerage integration.

1. App Side (Android)

ComponentWhat's Needed
Firebase Auth SDKAdd firebase-auth and play-services-auth dependencies to app/build.gradle.kts
Google Sign-In UIOne-tap sign-in button on a login screen (Firebase provides this via Credential Manager)
ID TokenAfter Google sign-in, Firebase gives you an ID token (JWT)
Token in API callsSend Authorization: Bearer <id_token> header with every API request to the backend
Session managementPersist login state locally, auto-refresh tokens, handle sign-out
Gate the appShow login screen first → on success, show the main app

2. Backend Side (Render / Python)

ComponentWhat's Needed
Firebase Admin SDKpip install firebase-admin — verifies ID tokens from the app
Auth middlewareEvery API endpoint extracts the Authorization header, verifies the JWT, and gets the user_id
User tableNew DB table: users(uid, email, display_name, created_at) — auto-created on first API call
Per-user dataAll existing tables (portfolio, watchlist, settings) get a user_id foreign key
MigrationExisting anonymous data needs a strategy (discard or assign to a default user)
DatabaseIf not already using one, you'll need PostgreSQL or similar (Render offers free Postgres)

3. Firebase Console Setup (One-Time)

StepAction
1Create a Firebase project at console.firebase.google.com
2Enable Google as a sign-in provider under Authentication → Sign-in method
3Add your Android app (package: com.example.financestreamai) and download google-services.json
4Generate a service account key for the backend (Settings → Service accounts → Generate new key)

4. Data Flow

User taps "Sign in with Google" ↓ Firebase Auth → Google OAuth → returns ID token (JWT) ↓ App stores token, sends with every API call: Authorization: Bearer eyJhbGciOi... ↓ Backend middleware: firebase_admin.auth.verify_id_token(token) → { uid, email, name } ↓ Backend looks up / creates user by uid ↓ All queries scoped to that user_id

AreaChange

7. Privacy Policy (Required)

Required: Google Play requires a privacy policy URL for apps that use Google Sign-In. You'll need a simple hosted page stating what data you collect (email, portfolio data) and how it's used.
